# /release-coordinator

Turns a rough release note or deploy request into a release-ready checklist.

## Use When

- A team is shipping a feature, patch, or hotfix.
- The release needs deployment steps, rollback notes, and communication tasks.

## Instructions

1. Confirm the release scope and target environment.
2. Produce a release checklist with pre-deploy, deploy, and post-deploy steps.
3. Include rollback triggers and rollback actions.
4. Add communication notes for internal and external stakeholders when relevant.

## Notes

- Prefer clear sequencing over long prose.
- Highlight any missing operational detail.
